# Metacognitive Monitor 设计

## 1. 目标

Metacognitive Monitor 的职责是让系统开始监控自己的认知质量，而不是只监控外部任务是否完成。

## 2. 当前检测内容

当前版本先检测三类风险：

- contradiction：当前请求是否与核心承诺或当前关注焦点冲突
- overload：输入复杂度是否高到需要更谨慎处理
- error_risk：综合估计当前任务的出错风险

## 3. 接入链路

当前监控器已接入在线任务链路：

1. 读取 prompt
2. 读取 Self Model 的关注焦点、核心承诺、已知限制
3. 生成元认知摘要
4. 将高风险告警注入 Global Workspace 信号池
5. 将元认知摘要返回给 API 调用方

## 4. 当前局限

当前是启发式版本，不具备：

- 长时推理轨迹审计
- 自我矛盾历史聚类
- 真实置信度校准
- 真实置信度校准仍然缺失

## 5. 下一阶段增强

- 将元认知结果写回 Self Model 快照
- 对“低置信度”自动触发更强的澄清策略
- 对“高矛盾”自动触发更完整的回滚与长期修复

## 6. 当前新增能力

当前版本已经接入基础反思回路：

1. 元认知监控器检测低置信度、高矛盾和过载
2. 反思引擎决定是否触发 reflection loop
3. 反思结果进入 workspace 信号池
4. 反思结果直接改变当前任务处理路径
