# 4.2 Development Plan

## Goal

Push the agent from policy-level self-evolution toward a controlled core-capability evolution pipeline that can generate, export, validate, and later train on high-quality offline data.

## Completed Today

1. Built a core-capability dataset export service.
2. Added API routes for dataset preview and export execution.
3. Added worker integration for asynchronous export jobs.
4. Added bundle writing for `manifest.json`, `sft.jsonl`, and `preference.jsonl`.
5. Reused active self-evolution policy when available and fell back to a recommended policy otherwise.
6. Extended evolution benchmarking to support baseline vs candidate comparison and legacy compatibility.
7. Upgraded the export manifest into a training-oriented manifest with dataset schemas and staged recommendations.
8. Connected real multi-turn dialogue traces into SFT export, instead of relying mainly on benchmark prompts.
9. Added offline export evaluation that scores dataset readiness before training.
10. Added controlled training job preparation that converts an export manifest into a local training job spec.
11. Added focused tests for dataset export, evaluation, training job preparation, and evolution flow.
12. Committed and pushed the initial export pipeline to `origin/main`.

## Validation

1. Focused test suite passed.
2. New export bundle test passed.
3. Export evaluation and training job preparation tests passed.
4. Git push eventually succeeded after repeated retries under unstable network conditions.

## Current Constraints

1. The system can now export, evaluate, and prepare training jobs, but it still does not execute real fine-tuning in-process.
2. Worker tasks are wired, but a full end-to-end always-on worker process still needs routine operational validation.
3. The local 0.5B model remains a hard ceiling on reasoning quality.
4. Local API startup on port 8000 has shown instability and should be debugged separately.

## Next Priority

1. Run a real asynchronous export/evaluation/training-prep sequence with a live worker and verify end-to-end artifact generation.
2. Add post-training evaluation hooks so the same export bundle can be used for before/after comparison.
3. Replace placeholder recommended training commands with actual local SFT and preference training entrypoints.
4. Expand multi-turn sample generation from recent dialogue into richer task-execution traces.
5. Debug local API startup instability on port 8000 so live console validation becomes reliable.

## Success Criteria

1. An agent can generate a reproducible training bundle from its own validated traces.
2. The exported bundle can be consumed by a downstream training script without manual restructuring.
3. Offline evaluation can compare pre-training and post-training behavior on the same benchmark/task set.
4. Core-capability evolution remains auditable, reversible, and bounded by explicit evaluation gates.
